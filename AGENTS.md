# Agent Guidance

## Flow Product Docs

Use Flow for lightweight product-development work in this workspace.

The Flow suite definition lives at:

```text
docs/flow-overview.md
```

Local specs and task breakdowns live under:

```text
specs/[feature]/prd.md
specs/[feature]/tasks/
```

## Writing Style

Write for PMs, designers, developers, clients, and stakeholders.

Prefer:

- Plain language
- Short sections
- Human-readable decisions
- Clear status, owner, reviewer, and assignee fields when useful

Avoid:

- Machine-readable schemas as the primary artifact
- Long agent-facing prose inside user-facing docs
- Overwriting existing specs or task files without explicit user approval
