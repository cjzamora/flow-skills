## What to Build

Create the shared foundation for the Flow skill suite so each individual skill follows the same product-development conventions, naming, writing style, folder structure, and safety rules.

This slice should make the suite recognizable and consistent before individual skills add their own behavior.

## Acceptance Criteria

- [ ] The Flow skill folders exist for `flow-scaffold`, `flow-grill`, `flow-spec`, `flow-design`, `flow-wireframe`, and `flow-handoff`.
- [ ] Each skill has a valid `SKILL.md` with name, description, and a concise purpose.
- [ ] Shared Flow conventions are documented once and referenced consistently by the skills.
- [ ] The suite uses `flow-*` naming and treats Flow as a plain workflow name, not an acronym.
- [ ] The skills instruct agents to write for PMs, designers, developers, clients, and stakeholders.
- [ ] The skills include overwrite protection for existing revisions.

## Blocked By

None - can start immediately.
